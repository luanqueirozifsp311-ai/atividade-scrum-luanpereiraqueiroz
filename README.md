# Site de Livraria

## Descrição

O sistema de Livraria será uma aplicação que permite cadastrar, visualizar e gerenciar livros disponíveis para venda.
Os usuários poderão buscar livros por título, autor ou categoria.
Também será possível controlar estoque e preços.
O objetivo é facilitar a organização e a venda de livros de forma simples e eficiente.

 ## Papéis do Scrum

- Product Owner (eu mesmo)
Responsável por definir o que deve ser feito no projeto, organizando e priorizando o Product Backlog.

- Scrum Master (eu mesmo)
Responsável por garantir que o Scrum está sendo aplicado corretamente e ajudar a remover impedimentos.

- Time de Desenvolvimento (eu mesmo)
Responsável por desenvolver o sistema, implementando as funcionalidades definidas.

## Eventos do Scrum

- Sprint
É um período de tempo fixo onde uma parte do projeto é desenvolvida.

- Sprint Planning
Reunião onde se decide o que será feito na Sprint.

- Daily Scrum
Reunião rápida diária para acompanhar o progresso e identificar problemas.

- Sprint Review
Reunião para apresentar o que foi desenvolvido na Sprint.

- Sprint Retrospective
Momento para refletir sobre o que pode ser melhorado no processo.

## Artefatos do Scrum

- Product Backlog
Lista de todas as funcionalidades que o sistema deve ter.

- Sprint Backlog
Conjunto de tarefas escolhidas para serem feitas na Sprint atual.

- Incremento
Resultado final da Sprint, ou seja, uma versão funcional do sistema.

## Por que essa ordem?
- As funcionalidades básicas vêm primeiro para garantir que o sistema funcione (cadastrar, listar e buscar livros).
- Depois vêm as funções de gerenciamento (editar, remover, estoque).
Por último, funcionalidades mais avançadas como compra e carrinho, pois dependem das anteriores.
